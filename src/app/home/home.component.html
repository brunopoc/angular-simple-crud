Instruções

  1. Utilize o github ou afins para versionar seu projeto.
  2. Você tem até 7 dias corridos para concluir as atividades. (O Resultado deverá ser enviado após os 7 dias, independente do ponto do desenvolvimento);
    1. Priorize 1 o em sua zona de conhecimento, e progrida o máximo que puder. Caso não consiga concluir todas as atividades, por favor, entregue o que foi feito até a data solicitada.
  3. Fique à vontade para utilizar tecnologias, frameworks e técnicas não citadas nas atividades.
  4. Recomendamos a utilização do docker (http://www.docker.com) para montagem do ambiente por facilitar a nossa execução e validação, mas fique à vontade para usar outras coisas, como cloud publica por exemplo, contanto que nós consigamos executar e validar sua infra/código.
    1. Caso opte pela utilização do docker, publique os Dockerfiles no repositório do projeto ou deixe a imagem publicada no Dockerhub.

Obs.: Para todos os itens iremos considerar a documentação como parte da entrega.

Case Lançamentos de Gastos Pessoais

  1. Crie interface de uma aplicação, para chamar as APIs descritas em https://github.com/adelbs/it-api-case
    1. Montar telas com operações do CRUD:
      1. Create – Cadastro de Categoria
      2. Read – Leitura de Categorias e Lançamentos
      3. Update – Atualização de Categoria
      4. Delete – Apagar Categoria
    2. Use Linguagem e Framework que desejar
    3. Colocar tratamento de erros nas telas
    4. Colocar consistências nos campos
    5. As telas precisam ser navegáveis entre si
    6. Montar bibliotecas comuns entre as telas
    7. Tela precisa ser responsiva
  2. Utilize técnicas ou descreva formas de aplicar os temas qualidade e teste na sua aplicação (teste unitário, teste regressivo, teste integrado, teste de performance, teste de resiliência etc)
  3. Publique o projeto no Github e documentar em um README.md os itens abaixo:
    1. Documentação do projeto
    2. Documentação de arquitetura
    3. Documentação de como podemos subir uma cópia deste ambiente localmente
  4. Itens avançados. Agora sua aplicação está em ambiente de Produção! Considere os itens abaixo na sua aplicação, ou apenas documente o que você faria para garantir cada tema:
    1. Escalabilidade
    2. Resiliência
    3. Alta disponibilidade
    4. Observabilidade – Monitoração e Log
    5. Segurança
    6. Acessibilidade
    7. Esteira CI/CD
